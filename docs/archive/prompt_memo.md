



Agent Chatting이랑 Knowledge Graph는 시냅스노트의 핵심 기능이야. 이 두 기능은 서로 유기적으로 연결되어야 해. Knowledge Graph에서 노드들을 컨텍스트를 자유롭게 클릭해서 컨텍스트로 가져올 수 있어야 하고, 에이전트 채팅의 내용을 적극적으로 노드로 만들 수 있어야 해.



Knowledge Graph는 각 노드의 디렉토리 depth를 기준으로, 클릭을 했을 때 해당 노드가 선택 상태가 되어 화면 중앙으로 오고, 해당 노드와 연결된 다른 노드들이 클릭하기 좋은 형태로 나타나야 해. 그리고 현재 선택된 노드와 연결된 노드들에 내용이 있는 경우 title과 함께 내용 preview가 살짝 나타나게 디자인하자.



노드는 내용이 없는 경우(concept 및 개념어) 원형으로 나타나게 하고, 내용이 있는 경우 모서리가 둥근 사각형. 사각형은 확장 / 축소 상태를 가져야 하고, 확장 상태일 때는 preview 일부를 나타내어야 함. 



노드는 단순 문서뿐만 아니라 디렉토리, 에이전트 채팅, 에이전트 생성 문서, 사람이 편집한 문서, 코드, Git Issue, 웹 article, PDF 문서 등 다양한 종류를 가져야 하며, 노드의 종류에 따라 직관적으로 해당 노드가 어떤 노드인지 확인할 수 있는 디자인을 가져야 함.



노드 그래프는 기본적으로 디렉토리 구조를 따르며, 루트 디렉토리부터 리프 노드까지 연결된 구조가 되어야 함.



노드들은 다중 선택 및 드래그 등을 통해 직관적으로 에이전트 채팅에 컨텍스트로 주입할 수 있어야 함. 

자 이제, 이 두 페이지를 기반으로 모바일 뷰를 만들자. 모바일 뷰에서는 좌우 스크롤을 통해 Chat / Context Manager / Knowledge Graph를 자연스럽게 전환할 수 있게 하는게 목표야

이 페이지 디자인을 기반으로 PDF, code, Git Issue, Directory, PDF 등 다양한 디자인의 노드를 추가하고, 왼쪽에는 컨텍스트 매니저가 있게끔 하자

문서 편집기 화면도 추가하자, 노드를 클릭해서 해당 노드 안으로 진입했을 때 나타나는 화면이어야 하고, 노션, 옵시디언 등에서 제공하는 마크다운 렌더링 및 문서 편집 기능을제공해야 해 

Chat / Context Manager / Knowledge Graph 이 세가지를 서로 수평적으로 연결된 구조로 만들자. Context Manager를 가운데에 두고, Chat+Context Manager가 있는 화면이 하나, Context Manager+Knowledge Graph가 있는 화면이 하나 있게끔 하고, Context Manager를 중심으로 둘 사이를 자유롭게 전환할 수 있게끔 하자. Chat은 현재 에이전트와의 채팅 내용을 나타내고, Knowledge Graph는 영구 지식 저장소를 나타내고, 그 사이에서 Context Manager가 브릿지 역할을 하는 거야.
