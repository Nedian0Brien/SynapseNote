# 그래프 뷰 기능 기획 로드맵

작성일: 2026-04-09
상태: Draft

## 배경

그래프 뷰는 SynapseNote의 핵심 시각화 화면으로, 사용자가 지식 구조를 탐색하고
노드 간 관계를 파악하는 주요 도구입니다. 현재는 기본적인 D3 기반 force simulation +
노드 선택 + 1-hop 하이라이트 + 사이드 패널(하단 독으로 전환됨) 수준이며,
이를 발전시킬 수 있는 여러 방향의 아이디어를 정리합니다.

## 그래프 뷰의 본질적 목적

사용자가 그래프 뷰에서 하는 활동은 크게 4가지입니다:

1. **지식 구조 탐색** — 전체 그래프를 훑으며 어떤 주제들이 있는지 파악
2. **관계 파악** — 특정 노드의 이웃과 연결 상태 확인
3. **경로 발견** — 두 지식이 어떻게 연결되는지 추적
4. **빠른 네비게이션** — 그래프를 통해 원하는 노트로 즉시 이동

모든 기능 개선은 이 4가지 활동을 더 잘 지원하도록 설계되어야 합니다.

---

## Category A — 정보 시각화 개선

### A1. 노드 크기와 색상의 의미 강화
- **Fresh glow**: 최근 24시간 이내 수정된 노드에 미묘한 펄스 애니메이션
- **미방문 노드 표시**: 아직 열어보지 않은 노드에 얇은 점선 테두리
- **태그 클러스터 색상**: 같은 태그를 공유하는 노드끼리 배경 halo 색조 공유

### A2. 엣지 타입의 시각적 구분 강화
- `wikilink`는 실선, `ref`는 점선, `directory`는 굵은 실선
- 방향성이 있는 엣지에 화살표 추가
- 호버 시 엣지 두께/불투명도로 강조

### A3. Minimap (미니맵)
우하단에 전체 그래프의 축소판을 작게 띄우고, 현재 뷰포트를 사각형으로 표시.
큰 그래프에서 "내가 어디 있는지" 파악 가능.

### A4. 시간축 타임라인 모드
하단에 얇은 타임라인 슬라이더. 드래그하면 그 시점까지 생성된 노드만 표시하는
"역사 재생" 모드. 지식 그래프의 성장 내러티브 시각화.

---

## Category B — 인터랙션 개선

### B1. Hover preview
노드 호버 시 0.5초 후 툴팁 페이드인 — 제목 + 요약 첫 줄 + 태그.
클릭 없이도 빠르게 훑어볼 수 있음.

### B2. Multi-select + 비교
Shift+Click으로 다중 선택 → 독이 "비교 모드"로 전환되어 공통 태그/공통 이웃 표시.

### B3. 경로 찾기 (Path finder)
두 노드 선택 → "경로" 버튼으로 최단 경로 하이라이트.
"어떻게 이 두 생각이 연결되는가?"를 시각적으로 답함.

### B4. Radial menu (원형 컨텍스트 메뉴)
우클릭 시 원형 메뉴가 펼쳐져 편집/삭제/포커스/복제 등 액션을 방사형 배치.
마우스 이동거리 최소화.

### B5. Keyboard navigation
- 방향키: 가장 가까운 노드로 이동
- `Tab`: 선택 노드의 다음 이웃으로 순환
- `/`: 검색바 포커스
- `Esc`: 선택 해제 + 전체 보기
- `F`: 선택 노드에 포커스

---

## Category C — 시각 효과 및 분위기

### C1. Particle flow on edges
선택 노드의 엣지를 따라 작은 파티클이 흐름. "데이터 흐름" 느낌.
절제해서 사용해야 고급스럽고, 과하면 장난감 느낌.

### C2. 배경 noise/grain texture
SVG filter로 아주 미묘한 noise 텍스처를 5% 미만 불투명도로 추가.
"깊이감 있는 공간" 느낌.

### C3. Depth of field (심도)
선택 안 된 노드를 `filter: blur(1px)`로 실제 블러 처리.
카메라 렌즈처럼 포커스 아웃 효과.

### C4. Cluster halo
같은 디렉터리에 속한 노드들 주위에 반투명한 convex hull 표시.
`d3-polygon`으로 구현 가능.

---

## Category D — 모바일/반응형

### D1. 독을 모바일에서는 bottom sheet로
화면이 좁을 때 독이 full-width로 바뀌고, 위로 드래그하면 상세 정보가 확장되는
iOS Apple Maps 스타일 bottom sheet.

### D2. 터치 제스처
- 한 손가락 드래그: 팬
- 두 손가락 핀치: 줌
- 두 손가락 탭: 전체 보기
- 길게 누르기: radial menu

---

## Category E — 네비게이션 개선

### E1. Breadcrumb trail
연속 탐색 시 상단에 "방문 경로"를 빵부스러기로 표시.
지식 탐색 세션의 맥락 유지.

### E2. Focus mode 토글
클릭 시 선택 노드 + 1-hop 이웃만 남기고 나머지를 숨김.
복잡한 그래프 집중 탐색.

### E3. Saved views
특정 시점의 줌/팬/필터 상태를 저장하고 복원.
"AI 연구" 뷰, "이번 주 작업" 뷰 등.

---

## Category F — 시각적 디테일

### F1. 노드 라벨 충돌 회피
`d3-force-label` 등으로 라벨끼리 밀어내기. 또는 줌 레벨 기반 밀도 표시/숨김.

### F2. Skeleton loading state
로딩 중 spinner 대신 노드 원들이 유령처럼 페이드인 되는 "건설 중" 애니메이션.

### F3. Empty state 디자인
노드가 없을 때 "첫 노트를 만들어보세요" 안내 + 일러스트.

---

## 우선순위 제안

### 1순위 (꼭 구현)
- **B1. Hover preview** — 정보 접근성 개선, 구현 간단
- **F1. 라벨 충돌 회피** — 가독성 필수
- **B5. Keyboard navigation** — 파워 유저 필수

### 2순위 (임팩트 큼)
- **A3. Minimap** — 큰 그래프 필수
- **E1. Breadcrumb trail** — 지식 탐색 기본
- **C3. Depth of field** — 시각 품질 향상

### 3순위 (차별화)
- **B3. Path finder** — 다른 그래프 도구와 차별화
- **A4. 타임라인 모드** — 독특한 경험
- **C4. Cluster halo** — 디렉터리 구조 명확성
